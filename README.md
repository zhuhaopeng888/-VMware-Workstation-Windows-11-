# -VMware-Workstation-Windows-11-
在 VMware Workstation 中扩大 Windows 11 虚拟机磁盘容量

**作者**：Judy / 原经验优化版  
**更新日期**：2026 年  
**适用版本**：VMware Workstation Pro / Player 16/17（含最新 17.x） + Windows 11（21H2 及以上）

本文详细记录了如何安全地将 VMware 虚拟机中的 Windows 11 系统磁盘容量扩大。整个过程分为 **VMware 层面扩展虚拟磁盘** 和 **Windows 内部扩展分区** 两大部分。

> **⚠️ 重要警告**
> - 操作前**必须备份虚拟机**：推荐直接复制整个虚拟机文件夹，或在 VMware 中创建快照（操作完成后可删除）。
> - 扩展磁盘前请确保虚拟机已**完全关机**（不能是挂起/暂停状态）。
> - 如果虚拟机存在快照，扩展按钮会变灰，需先删除所有快照。
> - 操作有一定风险，建议在非生产环境练习。

## 前提条件

- VMware Workstation 已安装并更新到最新版本。
- Windows 11 虚拟机已完全关机。
- 宿主机有足够剩余磁盘空间（新容量至少需要额外空间）。
- （推荐）已安装 VMware Tools（便于后续分辨率等适配）。

## 步骤一：在 VMware 中扩展虚拟磁盘（.vmdk）

1. **打开 VMware Workstation**，在左侧库中选中目标虚拟机。

2. **确保虚拟机已关机**。右键虚拟机 → **电源** → **关机客人操作系统**（或直接点击电源按钮关机）。

3. **编辑虚拟机设置**：
   - 右键虚拟机 → **编辑虚拟机设置**（或点击右侧的“编辑虚拟机设置”）。
   - 在 **硬件** 选项卡中找到 **硬盘**（通常是 SCSI 或 SATA 硬盘）并选中。

4. **扩展磁盘**：
   - 在右侧面板下方点击 **扩展...**（Expand...）按钮。
   - 输入新的目标容量（例如：从 60 GB 改为 120 GB）。
   - 建议选择 **将磁盘扩展为单个文件**（Single file），性能更好且管理方便。
   - 点击 **扩展**（Expand），等待进度完成。



5. **完成并保存**：点击 **确定** 保存设置。

**如果“扩展”按钮为灰色（不可用）**：

- 原因：存在快照（Snapshot）或链接克隆。
- 解决方法：
  1. 右键虚拟机 → **快照** → **快照管理器**。
  2. 删除所有快照（推荐“删除全部”）。
  3. 等待快照合并完成（时间取决于快照大小）。
  4. 再次尝试扩展。



## 步骤二：在 Windows 11 中扩展 C 盘分区

1. **启动虚拟机** 并正常登录 Windows 11。

2. **打开磁盘管理**：
   - 右键开始菜单（Windows 图标）→ **磁盘管理**。
   - 或按 `Win + R`，输入 `diskmgmt.msc` 后回车。

3. **查看磁盘状态**：
   - 你会看到 **磁盘 0**（系统盘）末尾出现一块黑色的 **未分配**（Unallocated）空间。



4. **直接扩展（理想情况）**：
   - 右键 **(C:) 分区** → **扩展卷**（Extend Volume）。
   - 跟随向导点击 **下一步** → **完成**。
   - 未分配空间会自动合并到 C 盘。

### 当“扩展卷”按钮灰色时（最常见情况）

原因：C 盘与未分配空间之间存在 **恢复分区**（Recovery Partition，通常 300~800 MB）或其他系统分区阻隔。Windows 自带磁盘管理只能合并**右侧紧邻**的未分配空间。

**推荐解决方案**（无损、安全）：

使用第三方免费分区工具移动/调整分区顺序。

#### 推荐工具（任选其一）

| 工具名称                  | 免费程度       | 优点                              | 下载建议                  |
|---------------------------|----------------|-----------------------------------|---------------------------|
| **DiskGenius**（首推）   | 免费版足够     | 操作直观、支持移动恢复分区、稳定 | 官网免费版                |
| **AOMEI Partition Assistant Standard** | 完全免费      | 界面友好、向导清晰                | 官网免费版                |

**使用 DiskGenius 示例步骤**（类似其他工具）：

1. 在虚拟机内下载并安装工具（建议从官网下载，避免捆绑软件）。
2. 打开 DiskGenius，选中磁盘 0。
3. 右键中间的 **恢复分区** → **调整/移动分区**（Resize/Move）。
4. 将恢复分区拖动到磁盘最右侧（紧靠未分配空间之后）。
5. 右键 **(C:) 分区** → **调整/移动分区**，向右扩展占用未分配空间。
6. 点击左上角 **应用**（Apply），确认执行操作。
7. 重启虚拟机后验证容量。



**不推荐但可行的命令行方法**（高风险，仅备份后使用）：

- 以管理员身份运行 CMD 或 PowerShell。
- 输入 `diskpart`。
- `list disk` → `select disk 0`。
- `list partition` 找到恢复分区编号。
- `select partition X`（X 为恢复分区号）。
- `delete partition override`。
- 退出 diskpart 后，在磁盘管理中即可直接扩展 C 盘。
- （可选）后续可使用工具重新创建恢复分区。

## 常见问题与解决方案

- **扩展后资源管理器仍显示旧容量**  
  → 重启虚拟机，或按 `F5` 刷新。此电脑 → 右键 C 盘 → 属性查看实际容量。

- **VMware 扩展按钮一直灰色**  
  → 确认已删除所有快照；检查是否为链接克隆（建议克隆为完整克隆再操作）。

- **扩展后蓝屏或启动失败**（极少见）  
  → 立即还原备份或快照；确保 VMware Tools 已更新。

- **想扩展多个分区**  
  → 同理使用分区工具调整顺序。

- **磁盘类型为 IDE vs SCSI**  
  → 现代 Windows 11 推荐使用 SCSI（LsiLogic 或 PVSCSI），性能更好。扩展前可转换为 SCSI（需注意驱动）。

## 推荐分区工具对比

- **DiskGenius**：界面简洁，移动分区成功率高，免费版已足够大多数虚拟机场景。
- **AOMEI**：向导式操作，非常适合新手。
- 避免使用未知来源工具，优先官网下载。

## 额外提示

- **性能优化**：扩展为单个文件后，可在 VMware 中运行 **VM → 管理 → 清理磁盘**（Clean up disks）进一步优化。
- **预防未来空间不足**：可为虚拟机额外添加一块独立数据盘（推荐），存放文档、下载等，避免 C 盘频繁膨胀。
- **最佳实践**：生产环境建议使用动态磁盘或单独数据盘；测试环境可大胆扩展。
- 操作完成后建议再次创建快照，作为新基准点。
